Part 7 · Chapter 39

The Agent SDK (Programmatic API)

Claude Code can be used not only as a CLI but as a programmatic library via the Agent SDK. This is how you embed Claude Code capabilities into your own applications.

39.1 SDK API Surface

// Single-shot query (V1 API — stable)
import { query } from '@anthropic-ai/claude-code'

const result = query({
  prompt: 'Fix the auth bug in login.ts',
  options: {
    model: 'claude-sonnet-4-20250514',
    maxTurns: 30,
    systemPrompt: 'You are a code assistant.',
    permissionMode: 'bypassPermissions',
    allowedTools: ['Read', 'Edit', 'Bash', 'Glob', 'Grep'],
    mcpServers: [{ command: 'node', args: ['my-server.js'] }],
    apiKey: process.env.ANTHROPIC_API_KEY,
  }
})

// Streaming: iterate over messages as they arrive
for await (const message of result) {
  if (message.type === 'text') console.log(message.text)
  if (message.type === 'tool_use') console.log(`Using: ${message.name}`)
  if (message.type === 'result') console.log(`Done: ${message.costUSD}`)
}

39.2 V2 Session API (Unstable)

import {
  unstable_v2_createSession,
  unstable_v2_resumeSession,
  unstable_v2_prompt
} from '@anthropic-ai/claude-code'

// Multi-turn conversation
const session = unstable_v2_createSession({
  cwd: '/path/to/project',
  model: 'claude-sonnet-4-20250514',
  maxTurns: 50,
})

// First message
for await (const msg of session.prompt('Read the README')) {
  console.log(msg)
}

// Follow-up (same session context)
for await (const msg of session.prompt('Now add a contributing guide')) {
  console.log(msg)
}

// One-shot convenience
const result = await unstable_v2_prompt('What files are here?', {
  model: 'claude-sonnet-4-20250514'
})

39.3 Custom MCP Tools (In-Process)

import { createSdkMcpServer, tool } from '@anthropic-ai/claude-code'
import { z } from 'zod'

// Define a custom tool
const myTool = tool(
  'get_weather',
  'Get current weather for a city',
  { city: z.string() },
  async ({ city }) => {
    const weather = await fetchWeather(city)
    return { content: [{ type: 'text', text: JSON.stringify(weather) }] }
  },
  { annotations: { readOnlyHint: true } }
)

// Create an in-process MCP server
const server = createSdkMcpServer({
  name: 'my-tools',
  version: '1.0.0',
  tools: [myTool],
})

// Use it with a query
const result = query({
  prompt: 'What is the weather in Tokyo?',
  options: {
    mcpServers: [server],  // In-process, no child process needed
  }
})

39.4 SDK Session Management

import { listSessions, getSessionMessages, getSessionInfo } from '@anthropic-ai/claude-code'

// List recent sessions
const sessions = await listSessions({ dir: '/path/to/project', limit: 50 })

// Read conversation history
const messages = await getSessionMessages(sessions[0].id, {
  includeSystemMessages: false,
  limit: 100,
})

// Get session metadata
const info = await getSessionInfo(sessions[0].id)

39.5 SDK Message Types

type SDKMessage =
  | { type: 'text', text: string }           // Streamed text output
  | { type: 'tool_use', name: string, input: object }  // Tool invocation
  | { type: 'tool_result', content: string }  // Tool result
  | { type: 'thinking', text: string }        // Extended thinking
  | { type: 'progress', data: object }        // Progress updates
  | { type: 'result', costUSD: number, usage: ModelUsage }  // Final result

type ModelUsage = {
  inputTokens: number
  outputTokens: number
  cacheReadInputTokens: number
  cacheCreationInputTokens: number
  webSearchRequests: number
  costUSD: number
  contextWindow: number
  maxOutputTokens: number
}

39.6 SDK Options

type SDKSessionOptions = {
  cwd?: string                    // Working directory
  model?: string                  // Model override
  maxTurns?: number               // Max API round-trips
  systemPrompt?: string           // Custom system prompt
  appendSystemPrompt?: string     // Append to default prompt
  permissionMode?: string         // 'default' | 'plan' | 'bypassPermissions'
  allowedTools?: string[]         // Tool whitelist
  disallowedTools?: string[]      // Tool blacklist
  mcpServers?: McpServerConfig[]  // MCP server configs
  verbose?: boolean               // Verbose output
  apiKey?: string                 // API key override
}

Lesson: The SDK API exposes the same query engine, tools, and permission system as the CLI. The difference is the interface: CLI uses React + Ink for rendering; SDK uses async generators for streaming. All harness engineering (permissions, context management, tool orchestration) is shared.
