Part 7 · Chapter 35

Extending the Agent (Adding New Tools)

35.1 Step-by-Step: Adding a New Tool

1. Create directory: src/tools/MyNewTool/

2. Create main file: MyNewTool.ts
   - Use buildTool() factory
   - Define inputSchema with Zod
   - Implement call()
   - Implement checkPermissions()
   - Set isConcurrencySafe() and isReadOnly()

3. Create UI: UI.tsx
   - renderToolUseMessage() — what the user sees when tool is invoked
   - renderToolResultMessage() — what the user sees when tool completes

4. Create prompt: prompt.ts
   - Return instruction text that's injected into the system prompt
   - Tell the model WHEN and HOW to use this tool

5. Register in src/tools.ts:
   - Add to getAllBaseTools() array
   - Optionally gate behind feature flag

6. Add smoke test:
   - Verify tool appears in getTools() output
   - Verify input schema is valid

35.2 Tool Implementation Checklist

□ Name is unique across all tools (including MCP tools)
□ Input schema validates all parameters
□ checkPermissions() returns appropriate behavior
□ isConcurrencySafe() is correct (default: false = safe)
□ isReadOnly() is correct (affects permission decisions)
□ maxResultSizeChars is set appropriately
□ shouldDefer is set if tool is rarely needed
□ prompt() gives clear instructions with examples
□ Error cases return ToolResult with is_error, not throw
□ File operations update readFileState cache
□ Long operations use onProgress for progress reporting

35.3 Adding a New Built-In Agent

// src/tools/AgentTool/built-in/myAgent.ts
export const myAgent: BuiltInAgentDefinition = {
  agentType: 'my-agent',
  whenToUse: 'Use when the user needs X',

  getSystemPrompt(context) {
    return `
      You are a specialist in X.
      Your role is to...

      You have access to these tools: ${context.tools.map(t => t.name).join(', ')}

      === CONSTRAINTS ===
      - Always do A before B
      - Never do C without asking
    `
  },

  tools: ['Read', 'Glob', 'Grep', 'Bash'],  // Subset of tools
  disallowedTools: ['Agent'],  // Prevent recursive spawning
  model: undefined,  // Inherit from parent
  maxTurns: 30,
  permissionMode: 'default',

  memory: {
    user: true,     // Load user-level memory
    project: true,  // Load project-level memory
    local: false,   // No machine-specific memory
  }
}

35.4 Adding a New Hook Event

// In settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "FileEdit",
        "command": "npx prettier --write \"$CLAUDE_TOOL_INPUT_FILE_PATH\"",
        "timeout": 10000
      }
    ]
  }
}

Hook receives environment variables:

  • CLAUDE_SESSION_ID — current session
  • CLAUDE_TOOL_NAME — which tool was used
  • CLAUDE_TOOL_INPUT — JSON of tool input
  • CLAUDE_FILE_PATH — affected file (if applicable)
  • CLAUDE_ENV_FILE — temp file to write env vars back
