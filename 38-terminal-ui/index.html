Terminal UI Engineering (React + Ink)

Claude Code’s terminal UI is built with React + Ink, treating the terminal as a render target just like the browser DOM. This is a powerful pattern for building rich terminal applications.

38.1 Why React for the Terminal?

The traditional approach to terminal UIs is imperative: manually write ANSI escape codes, track cursor position, and handle redraws. React + Ink makes this declarative:

Traditional:                          React + Ink:
process.stdout.write('\x1b[2J')       function App() {
process.stdout.write('\x1b[H')         return (
process.stdout.write('Loading...')        <Box flexDirection="column">
// Manually track state                    <Text color="green">Loading...</Text>
// Manually handle redraws                 <Spinner />
                                          </Box>
                                        )
                                      }

38.3 Custom React Reconciler

Claude Code implements a full React reconciler — the same interface that React DOM and React Native implement:

// The reconciler translates React operations to terminal operations:
const reconciler = createReconciler({
  // Create a terminal DOM node
  createInstance(type, props) {
    return new TerminalNode(type, props)
  },

  // Append child to parent
  appendChild(parent, child) {
    parent.children.push(child)
  },

  // Commit changes to terminal
  commitUpdate(node, updatePayload, type, oldProps, newProps) {
    node.props = newProps
    // Trigger re-render to terminal
  },

  // Handle text content
  createTextInstance(text) {
    return new TextNode(text)
  }
})

38.5 Hooks for Terminal Interaction (~80 hooks)

// Input handling
useTextInput()          // Text input with cursor, history
useVimInput()           // Vim-mode keybindings
usePasteHandler()       // Multi-line paste detection
useInputBuffer()        // Buffered keystroke handling

// Permission system
useCanUseTool()         // Permission check hook
usePermissionRequest()  // Permission dialog state

// IDE integration
useIDEIntegration()     // VS Code/JetBrains bridge
useDiffInIDE()          // Open diffs in IDE

// Session management
useSessionBackgrounding()  // Background session handling
useRemoteSession()         // Remote session connection
useAssistantHistory()      // Conversation history navigation

38.6 Why This Matters for Agent Builders

Even if you don’t use React + Ink, the patterns are universal:

  1. Declarative UI updates — Don’t manually manage terminal state. Use a rendering layer.
  2. Component composition — Permission dialogs, tool results, and messages are all components that can be composed.
  3. State-driven rendering — The UI reflects the state store. When state changes, the UI updates automatically.
  4. Hooks for side effects — Use the hooks pattern for input handling, session management, and IDE integration.
