Plugin Architecture

Plugins extend Claude Code’s capabilities with new tools, commands, and prompts.

16.1 Plugin Structure

type Plugin = {
  name: string
  version: string
  description: string

  // Extensions
  tools?: ToolDef[]
  commands?: Command[]
  hooks?: HookConfig
  mcpServers?: MCPServerConfig[]
  skills?: SkillDefinition[]

  // Lifecycle
  onLoad?(context: PluginContext): Promise<void>
  onUnload?(): Promise<void>
}

16.2 Plugin Lifecycle

1. Discovery — scan plugin directories and marketplace
       │
       ▼
2. Installation — /plugin command or auto-install
       │
       ▼
3. Loading — initialize at startup or on-demand
       │
       ▼
4. Registration — add tools, commands, hooks to registries
       │
       ▼
5. Execution — plugins contribute to agent capabilities
       │
       ▼
6. Auto-update — check for updates in background
