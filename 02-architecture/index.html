Startup Sequence

Claude Code’s startup is designed for speed through parallel prefetching:

1. Fire side-effects in parallel:
   ├── MDM policy reads
   ├── Keychain prefetch
   ├── API preconnect
   └── Feature flag checks

2. Parse CLI arguments (Commander.js)

3. Initialize core systems:
   ├── Config loading & schema validation
   ├── Telemetry/analytics
   ├── OAuth/auth resolution
   └── MCP server connections

4. Launch React/Ink renderer

5. Enter REPL loop

Lesson: Heavy initialization should be parallelized. Don’t wait for MDM policy checks before starting to parse CLI arguments.
