Speculation & Prefetch (Parallel Execution)

Speculation is one of Claude Code’s most advanced features: pre-executing the next suggested action while the user is still reading the current output.

27.1 How Speculation Works

Model completes response
         │
         ├──► Display to user
         │
         └──► Generate suggested next action
                    │
                    ▼
              Start speculation
              (overlay filesystem)
                    │
                    ▼
         ┌──────────────────────┐
         │  Speculated Agent     │
         │  (copy-on-write)      │
         │                      │
         │  Reads: real files    │
         │  Writes: overlay dir  │
         │  Bash: read-only only │
         └─────────┬────────────┘
                   │
         ┌─────────┴─────────┐
         │                   │
    User accepts         User rejects
         │                   │
         ▼                   ▼
    Copy overlay         Delete overlay
    to real FS           (no changes)
         │
         ▼
    Inject messages
    into conversation

27.2 Copy-on-Write Isolation

function startSpeculation(suggestionText, context) {
  // 1. Create overlay directory
  const overlayDir = `~/.claude/temp/speculation/${pid}/${id}/`

  // 2. Spawn forked agent with permission boundaries
  const result = await runForkedAgent({
    canUseTool: (tool, input) => {
      // Write tools → write to overlay, not real FS
      if (isWriteTool(tool)) {
        if (permissionAllows(tool)) {
          redirectToOverlay(input.path, overlayDir)
          return true
        }
        return false  // Boundary: speculation stops
      }

      // Read tools → read from overlay if written, else real FS
      if (isReadOnlyTool(tool)) {
        return true
      }

      // Bash → only read-only commands
      if (tool === 'Bash') {
        return isReadOnlyBashCommand(input.command)
      }

      // All other tools → denied (speculation stops)
      return false
    }
  })
}

27.3 Pipelined Speculation

Suggestion 1 → Speculation 1 → Complete → Generate Suggestion 2
                                              │
                                              ▼
                                    Speculation 2 (starts immediately)
                                              │
User accepts Suggestion 1                     │
         │                                    │
         ▼                                    ▼
Apply Speculation 1 results        Speculation 2 continues...

Lesson: Speculation eliminates wait time for the user. The key insight is copy-on-write isolation — speculated changes never touch the real filesystem until accepted. If the user doesn’t accept, the overlay is simply deleted with zero cost.

27.4 Acceptance Flow

async function acceptSpeculation(state) {
  // 1. Abort active speculation
  state.abort()

  // 2. Copy written files from overlay to main filesystem
  for (const path of state.writtenPaths) {
    copyFromOverlay(path, overlayDir, realFS)
  }

  // 3. Clean up overlay
  deleteDirectory(overlayDir)

  // 4. Inject speculated messages into conversation
  const messages = prepareMessagesForInjection(state.messages)
  // Strip: failed tool_uses, thinking blocks, interrupts
  // Keep: successful tool_use/tool_result pairs

  return { messages, timeSavedMs: elapsed }
}
