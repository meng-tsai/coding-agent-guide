Observability & Debugging Agents

34.1 Analytics Event System

// Public API for event logging
logEvent('tengu_prompt_cache_break', {
  reason: 'tools_changed',
  addedTools: ['NewTool'],
  removedTools: [],
  tokenDrop: 5000,
  timeSinceLastMsg: 120000
})

// Metadata safety types prevent accidental PII leaks
type SafeMetadata = AnalyticsMetadata_I_VERIFIED_THIS_IS_NOT_CODE_OR_FILEPATHS
type PIIMetadata = AnalyticsMetadata_I_VERIFIED_THIS_IS_PII_TAGGED

// Event routing: Datadog (general, stripped of PII) + 1P (privileged PII columns)
// Sampling via shouldSampleEvent() for high-frequency events

34.2 Key Events to Track

EventWhat It Tells You
prompt_cache_breakSystem prompt/tools changed, cache invalidated (cost spike)
compact_failedContext window management broke
compact_ptl_retryPrompt too long, forced to truncate
tool_permission_deniedUser denied an operation (model may be stuck)
advisor_tool_token_usageNested tool cost accumulation
fork_agent_querySub-agent spawned with usage metrics
coordinator_mode_switchedSession mode changed mid-conversation

34.4 Prompt Cache Break Detection

This is critical for cost optimization:

// A cache break means the API can't reuse cached prompt prefix
// → Input tokens jump from cache price to full price (5-10x more expensive)

// Detection: >5% drop in cache_read_tokens AND >2000 absolute drop
if (cacheDrop > 0.05 && absoluteDrop > 2000) {
  // Log exactly what changed:
  // - System prompt hash changed?
  // - Tool schemas changed? Which ones?
  // - Model changed?
  // - Cache TTL expired? (5min or 1hour)
}

Lesson: Without cache break detection, you won’t know why your costs suddenly spike. Track it from day one.

34.5 Circuit Breakers

// Auto-compact circuit breaker
if (consecutiveCompactFailures >= 3) {
  stopAutoCompact()  // Requires manual intervention
}

// Denial tracking circuit breaker
if (toolDenialCount > threshold) {
  escalateToUser()  // Model is stuck in a loop
}
