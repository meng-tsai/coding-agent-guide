Part 2 · Chapter 8

Multi-Agent Collaboration

Claude Code supports spawning sub-agents for complex tasks, with sophisticated isolation and communication patterns.

8.1 Agent Definition

type AgentDefinition = {
  agentType: string              // 'general-purpose', 'plan', 'explore', etc.
  whenToUse: string              // Description for the model
  tools?: string[]               // Allowed tools (subset)
  disallowedTools?: string[]     // Explicitly blocked tools
  skills?: string[]              // Available skills
  model?: string                 // Model override
  effort?: string                // Reasoning effort level
  permissionMode?: string        // Permission mode override
  maxTurns?: number              // Maximum conversation turns
  memory?: {                     // Memory scoping
    user?: boolean
    project?: boolean
    local?: boolean
  }
  hooks?: HookConfig             // Agent-specific hooks
  mcpServers?: string[]          // Available MCP servers
  isolation?: 'worktree' | 'remote'  // Execution isolation
}

8.3 Agent Spawning

// AgentTool spawns a sub-agent
async function runAgent(definition, prompt, context) {
  // 1. Create isolated context
  const agentContext = createAgentContext(definition, context)

  // 2. Build agent-specific system prompt
  const systemPrompt = buildEffectiveSystemPrompt({
    agent: definition,
    mode: definition.permissionMode
  })

  // 3. Filter tools based on agent definition
  const tools = filterToolsForAgent(definition, context.options.tools)

  // 4. Create a new QueryEngine instance
  const engine = new QueryEngine({
    systemPrompt,
    tools,
    model: definition.model,
    maxTurns: definition.maxTurns
  })

  // 5. Execute the query
  const result = await engine.submitMessage(prompt)

  // 6. Return summarized result to parent
  return { data: result.summary }
}

8.4 Agent Isolation Modes

Worktree Isolation

Parent Agent (main branch)
    │
    ├──► create git worktree ──► Sub-Agent (isolated copy)
    │                                │
    │                                ├── Free to modify files
    │                                ├── Can commit changes
    │                                └── Changes don't affect parent
    │
    └──► merge or discard worktree

Remote Isolation

Parent Agent (local machine)
    │
    └──► spawn remote agent ──► Sub-Agent (remote machine)
                                    │
                                    ├── Isolated execution
                                    ├── Results sent back
                                    └── Cleanup on completion

8.5 Inter-Agent Communication

// SendMessageTool enables agent-to-agent messaging
const SendMessageTool = buildTool({
  name: 'SendMessage',
  async call({ to, message }, context) {
    // Send message to specified agent
    const targetAgent = context.getAppState().teammates.find(a => a.id === to)
    if (!targetAgent) throw new Error(`Agent ${to} not found`)

    await targetAgent.receiveMessage(message)
    return { data: { sent: true } }
  }
})

8.6 Coordinator Mode

For complex multi-agent workflows:

┌─────────────────────────────────────────────┐
│              Coordinator Agent               │
│  (Specialized system prompt)                 │
│                                             │
│  Only has access to:                         │
│  - AgentTool (spawn workers)                │
│  - TaskCreateTool (track work)              │
│  - SendMessageTool (communicate)            │
│  - TaskUpdateTool (status updates)          │
│                                             │
│  Does NOT have:                              │
│  - BashTool, FileEditTool, etc.             │
│  - Cannot directly modify the environment    │
└─────────────┬───────────────────────────────┘
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
 Worker    Worker    Worker
 Agent     Agent     Agent
 (full     (full     (full
  tools)    tools)    tools)

Lesson: The coordinator should be unable to directly modify the environment. Its only power is orchestrating other agents. This is the principle of least privilege applied to multi-agent systems.

8.7 Agent Tool Restrictions

// Different agent contexts get different tool subsets
const ALL_AGENT_DISALLOWED_TOOLS = [
  // Prevent recursion
  'EnterPlanMode', 'ExitPlanMode',
  // Prevent scope leaks
  'EnterWorktree', 'ExitWorktree',
]

const ASYNC_AGENT_ALLOWED_TOOLS = [
  // Background agents only get safe tools
  'Read', 'Glob', 'Grep', 'WebFetch', 'WebSearch',
  'TaskCreate', 'TaskUpdate', 'TaskGet', 'TaskList',
]

const COORDINATOR_MODE_ALLOWED_TOOLS = [
  // Coordinator only gets orchestration tools
  'Agent', 'SendMessage', 'TaskCreate', 'TaskUpdate',
  'TaskGet', 'TaskList', 'TaskStop',
]
