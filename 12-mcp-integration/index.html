Part 3 · Chapter 12

MCP Integration (Model Context Protocol)

MCP allows the agent to consume tools and resources from external servers, and expose its own capabilities as an MCP server.

12.1 MCP Client Architecture

┌──────────────────┐     ┌──────────────────────┐
│   Claude Code    │     │   MCP Server A        │
│   (MCP Client)   │◄───►│   (Custom tools)      │
│                  │     └──────────────────────┘
│  - Discovers     │     ┌──────────────────────┐
│    tools/resources│◄───►│   MCP Server B        │
│  - Invokes tools │     │   (Custom resources)  │
│  - Reads resources│     └──────────────────────┘
│  - Handles auth  │     ┌──────────────────────┐
└──────────────────┘◄───►│   MCP Server C        │
                         │   (External API)      │
                         └──────────────────────┘

12.2 MCP Configuration

// .mcp.json
{
  "mcpServers": {
    "my-server": {
      "command": "node",
      "args": ["path/to/server.js"],
      "env": {
        "API_KEY": "..."
      }
    }
  }
}

12.3 MCP Tool Wrapping

// MCP tools are wrapped as native tools with a prefix:
// mcp__serverName__toolName

const MCPTool = buildTool({
  name: `mcp__${serverName}__${toolName}`,
  isMcp: true,
  mcpInfo: { serverName, toolName },

  async call(input, context) {
    const client = context.options.mcpClients.find(c => c.name === serverName)
    const result = await client.callTool(toolName, input)
    return { data: result }
  }
})

12.4 MCP Server Mode

Claude Code can also run as an MCP server:

// src/entrypoints/mcp.ts
const server = new McpServer({
  tools: {
    list_tools: { /* ... */ },
    read_source_file: { /* ... */ },
    search_source: { /* ... */ },
    get_architecture: { /* ... */ },
  },
  resources: {
    'claude-code://architecture': { /* ... */ },
    'claude-code://tools': { /* ... */ },
    'claude-code://source/{path}': { /* ... */ },
  },
  prompts: {
    explain_tool: { /* ... */ },
    architecture_overview: { /* ... */ },
  }
})

12.5 MCP Server Instructions in System Prompt

When MCP servers provide instructions, they’re injected as system reminders:

<system-reminder>
# MCP Server Instructions
### my-database-server
Use this server to query the production database. Always use parameterized
queries. Never run DELETE without a WHERE clause.
</system-reminder>
