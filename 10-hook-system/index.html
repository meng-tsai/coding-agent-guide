Hook System & Lifecycle Events

The hook system allows extending agent behavior at well-defined lifecycle points without modifying core code.

10.1 Hook Events

10.2 Hook Configuration

// In settings.json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "command": "echo 'About to run bash command'",
        "timeout": 5000
      }
    ],
    "PostToolUse": [
      {
        "matcher": "FileEdit",
        "command": "npx prettier --write $CLAUDE_FILE_PATH",
        "timeout": 10000
      }
    ],
    "UserPromptSubmit": [
      {
        "command": "echo 'User submitted: $CLAUDE_USER_PROMPT'",
        "timeout": 3000
      }
    ]
  }
}

10.3 Hook Matchers

// Glob patterns for matching tool invocations
"Bash"                  // Match all Bash tool calls
"Bash(git *)"           // Match Bash calls starting with 'git'
"FileEdit(/src/*)"      // Match edits to files in src/
"skill/my-skill"        // Match specific skill invocations

10.4 Hook Execution

async function executeHook(hook, event, signal) {
  // 1. Setup environment
  const env = {
    ...process.env,
    CLAUDE_SESSION_ID: sessionId,
    CLAUDE_TOOL_NAME: event.toolName,
    CLAUDE_TOOL_INPUT: JSON.stringify(event.input),
    CLAUDE_FILE_PATH: event.filePath,
    CLAUDE_ENV_FILE: tempEnvFile,  // Hook can write env vars here
  }

  // 2. Spawn subprocess
  const child = spawn('sh', ['-c', hook.command], { env, timeout: hook.timeout })

  // 3. Parse JSON output
  const output = await readOutput(child)
  const result = JSON.parse(output)

  // 4. Process hook decisions
  if (result.permissionDecision) {
    return { permissionDecision: result.permissionDecision }
  }
  if (result.modifiedOutput) {
    return { modifiedOutput: result.modifiedOutput }
  }

  return {}
}

10.5 Session Hooks (In-Memory)

// Ephemeral hooks registered by agents at runtime
const sessionHooks = new Map<string, HookCallback>()

function registerSessionHook(event, callback) {
  sessionHooks.set(event, callback)
  // Automatically cleaned up at session end
}

Lesson: Two hook types serve different needs: command hooks (shell subprocesses) for user configuration, and session hooks (in-memory callbacks) for runtime agent behavior. Both share the same event system.
