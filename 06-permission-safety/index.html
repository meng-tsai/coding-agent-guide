Permission & Safety Model

The permission system is Claude Code’s most sophisticated subsystem (~25K lines). It ensures the agent never performs an action the user hasn’t authorized.

6.1 Permission Modes

ModeBehaviorUse Case
defaultPrompt user for each potentially destructive operationNormal interactive use
planShow full execution plan, ask once for batch approvalComplex multi-step tasks
bypassPermissionsAuto-approve everythingTrusted CI environments
autoML-based classifier decidesExperimental advanced mode

6.2 Permission Rule System

Rules use wildcard patterns:

# Allow rules (auto-approve matching operations)
Bash(git *)           # Allow all git commands
Bash(npm test)        # Allow specific npm command
FileEdit(/src/*)      # Allow edits within src/
FileRead(*)           # Allow reading any file

# Deny rules (block matching operations)
Bash(rm -rf *)        # Block recursive force delete
FileEdit(/.env)       # Block editing environment files

6.3 Permission Resolution Order

1. Deny Always rules     → DENY  (highest priority — block no matter what)
2. Allow Always rules    → ALLOW
3. Ask If Match rules    → ASK   (prompt user)
4. Pre-Tool-Use hook     → Hook can override with permissionDecision
5. ML Classifier (auto)  → Classifier decides
6. Default mode behavior → Based on permission mode

6.4 Permission Flow

async function hasPermissionsToUseTool(tool, input, context): Promise<PermissionResult> {
  // 1. Check deny rules
  const denyRule = getDenyRuleForTool(context, tool)
  if (denyRule) return { behavior: 'deny', reason: denyRule }

  // 2. Check tool's own permission check
  const toolResult = await tool.checkPermissions(input, context)
  if (toolResult.behavior !== 'allow') return toolResult

  // 3. Check allow rules
  const allowRule = getAllowRuleForTool(context, tool, input)
  if (allowRule) return { behavior: 'allow' }

  // 4. Check ask rules
  const askRule = getAskRuleForTool(context, tool, input)
  if (askRule) return { behavior: 'ask', reason: askRule }

  // 5. Run pre-tool-use hook
  const hookResult = await runPreToolUseHook(tool, input)
  if (hookResult?.permissionDecision) return hookResult.permissionDecision

  // 6. Auto-mode classifier
  if (context.mode === 'auto') {
    return await runAutoClassifier(tool, input)
  }

  // 7. Default: ask user
  return { behavior: 'ask', reason: 'No matching rule' }
}

6.5 BashTool Permission Matcher

The BashTool has special permission handling because shell commands can be compound:

// "git push && rm -rf /" is TWO commands
// The permission matcher parses them individually

async preparePermissionMatcher({ command }) {
  const parsed = await parseForSecurity(command)

  if (parsed.kind !== 'simple') {
    return () => true  // Fail-safe: complex commands require permission
  }

  const subcommands = parsed.commands.map(c => c.argv.join(' '))
  return pattern => subcommands.some(cmd => matchWildcardPattern(pattern, cmd))
}

Lesson: Compound shell commands must be decomposed for accurate permission matching. git status && rm -rf / should match the rm deny rule even though it starts with git.

6.6 Denial Tracking

// Track cumulative denials per tool
type DenialTrackingState = {
  denialCounts: Map<string, number>
  lastDenialTimestamp: Map<string, number>
}

// After threshold denials, fall back to always-ask
if (denialCount > THRESHOLD) {
  // Model keeps trying denied operations — escalate to user
  return { behavior: 'ask', reason: 'Repeated denials' }
}

Lesson: If the model keeps trying operations that get denied, the system should escalate rather than silently denying forever. This prevents the model from getting stuck in a loop.

6.7 Auto-Mode Classifier

// ML-based permission classifier
// Trained to distinguish safe from dangerous operations
async function runAutoClassifier(tool, input): Promise<PermissionResult> {
  const classifierInput = tool.toAutoClassifierInput(input)
  const decision = await classify(classifierInput)

  if (decision.confidence > THRESHOLD) {
    return { behavior: decision.allow ? 'allow' : 'deny' }
  }

  // Low confidence: fall back to asking user
  return { behavior: 'ask', reason: 'Classifier uncertain' }
}
