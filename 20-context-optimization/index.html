Context Window Optimization (Compact)

When conversations get long, the context window fills up. The compact service manages this.

20.1 Compact Strategies

StrategyTriggerAction
Auto-CompactToken count > thresholdSummarize old messages
Reactive CompactNear context limitAggressive summarization
Snip CompactHISTORY_SNIP featureRemove middle of conversation
Context CollapseTool results too largeCollapse verbose results

20.2 Auto-Compact Flow

Conversation messages: [M1, M2, M3, ... M50]
                                │
                                ▼
Token estimate exceeds COMPACT_THRESHOLD
                                │
                                ▼
Select messages to compact (oldest first, keep recent)
                                │
                                ▼
Send to LLM: "Summarize these messages into key facts"
                                │
                                ▼
Replace old messages with compact summary
                                │
                                ▼
New messages: [Summary, M45, M46, M47, M48, M49, M50]

20.3 Context Collapse

// Large tool results are collapsed to save context
function collapseToolResult(result, maxChars) {
  if (result.length <= maxChars) return result

  return [
    result.slice(0, maxChars / 2),
    '\n\n[... collapsed ...]\n\n',
    result.slice(-maxChars / 4)
  ].join('')
}

20.4 Post-Compact Hooks

// Hooks run after compaction
hooks.PostCompact.forEach(hook => {
  hook.execute({
    compactedMessageCount: oldMessages.length,
    newSummaryLength: summary.length
  })
})
